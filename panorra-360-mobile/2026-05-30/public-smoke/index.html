Issue #2615: Set Up Project Folder Structure (pages-based architecture)

Author: anurdin01 | Updated: 2026-05-26T08:03:48Z

Type

Acceptance Criteria

  • All folders and placeholder index.ts / index.tsx files are created per the structure above
  • Each page folder (home, place-detail, post-detail) has components/, hooks/, store/, types/, and index.tsx
  • shared/components/, shared/hooks/, shared/utils/ are created
  • api/ folder has client.ts scaffold and types/index.ts
  • theme/ folder is created (tokens populated in #2588)
  • navigation/index.tsx scaffold is created
  • TypeScript path aliases are configured (e.g., @pages/, @shared/, @api/, @theme/) in tsconfig.json and bundler config
  • Existing code is migrated or removed to match the new structure without breaking the build

Dependencies

None — this should be the first task executed before any other slicing or integration work.

Folder Location

Author: anurdin01 | Updated: 2026-05-26T08:34:17Z

Type

Integration

Overview

Define the global Zustand store slices needed across all modules: homeStore (map state, feed state, search state), placeStore (active place, tabs), and postStore (active post, viewer state). Use Zustand's create with TypeScript-typed state and actions, separated into slice files combined into a root store.

Acceptance Criteria

  • Three store slices exist: homeStore, placeStore, postStore
  • Each slice has clearly typed State and Actions interfaces
  • Stores are initialized with sensible default values
  • No circular dependencies between slices
  • Stores are accessible via custom hooks (e.g., useHomeStore, usePlaceStore, usePostStore)

Issue #2593: Implement MapLibre Map Base Screen

Author: anurdin01 | Updated: 2026-05-29T10:15:04Z

Type

Slicing

Overview

Render the MapLibre map as the full-screen base layer of the Home screen. Configure initial camera position, map style, and zoom constraints. The map should fill the screen behind the Header and any overlaying UI components.

Map Embed URL

Use WebView embed instead of native MapLibre SDK.

  • Live: https://maps.panorra.com/embed/panorra?d=43.6532,-79.3832,9.791463986843617&showLocation=true&forceMobile=true&allow-overlap-at=12
  • Dev: https://maps.panorra.com/development/embed/panorra?d=43.6532,-79.3832,9.791463986843617&showLocation=true&forceMobile=true&allow-overlap-at=12

Select URL based on env.baseURLType ('live' vs 'development').

Acceptance Criteria

  • MapLibre map renders full-screen on both Android and iOS
  • Initial map region matches the design (city-level zoom)
  • Map embed URL switches between live/dev via environment variable
  • Map gestures (pan, pinch-to-zoom) work correctly
  • Header component sits above the map without clipping
